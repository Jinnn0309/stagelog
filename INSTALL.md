# StageLog 微信小程序安装指南

## 快速开始

### 1. 环境准备
- 下载并安装微信开发者工具
- 注册微信小程序账号（可选）

### 2. 导入项目
1. 打开微信开发者工具
2. 选择"导入项目"
3. 项目目录选择：`/Users/jinnn/Downloads/stagelog/src`
4. AppID可以选择"测试号"或填写自己的AppID
5. 项目名称：StageLog
6. 点击"导入"

### 3. 运行项目
- 导入成功后项目会自动编译
- 在模拟器中可以看到效果
- 可以在真机上预览（需要扫码）

## 常见问题解决

### 问题1：项目无法编译
**可能原因**：缺少必要的文件或配置错误
**解决方案**：
1. 检查 `src` 目录是否包含完整的项目结构
2. 确认所有 `.js`, `.json`, `.wxml`, `.wxss` 文件都存在
3. 检查 `app.json` 中的页面路径是否正确

### 问题2：TabBar不显示
**可能原因**：图标文件缺失
**解决方案**：
- 当前版本已移除图标依赖，使用文字TabBar
- 如果需要图标，可以自行添加 `images/` 目录下的图标文件

### 问题3：页面跳转失败
**可能原因**：路径配置错误
**解决方案**：
- 检查 `app.json` 中 `pages` 数组配置
- 确认每个页面文件夹下都有对应的4个文件

### 问题4：样式显示异常
**可能原因**：WXSS语法问题
**解决方案**：
- 检查是否有不支持的CSS属性
- 使用微信开发者工具的调试工具检查样式

### 问题5：数据不保存
**正常现象**：小程序使用本地存储
**说明**：
- 数据保存在小程序本地存储中
- 换设备或清除缓存会丢失数据
- 可以添加云端存储功能

## 开发说明

### 修改代码
1. 直接修改对应的 `.js` 或 `.wxml` 文件
2. 保存后微信开发者工具会自动重新编译
3. 在模拟器中查看效果

### 调试技巧
1. 使用 `console.log()` 在开发者工具控制台查看输出
2. 使用断点调试功能
3. 查看网络请求（如果有）
4. 检查存储数据（Storage面板）

### 添加新页面
1. 在 `pages/` 目录下创建新文件夹
2. 创建对应的4个文件：`.js`, `.json`, `.wxml`, `.wxss`
3. 在 `app.json` 的 `pages` 数组中添加页面路径
